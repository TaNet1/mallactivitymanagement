# 商场活动管理 · 交互说明

> 适用页面：商场活动管理（`商场活动管理` 导航项）
> 对应代码：`src/views/EventManage.vue` 及 `src/components/*`
> 文档目的：说明页面的区域划分、字段规则、各类操作的交互行为与边界条件，供产品 / 前端 / 测试对齐。

---

## 1. 页面整体结构

页面自左到右分为三个固定区域：

| 区域 | 说明 | 对应组件 |
| --- | --- | --- |
| ① 左侧主导航 | 全局菜单，静态展示，当前高亮「商场活动管理」 | `Sidebar.vue` |
| ② 模块 Tab 操作栏 | 管理活动标签页（增 / 删 / 选 / 排序） | `LeftPanel.vue` |
| ③ 内容编辑区 | 顶部语言切换 + 保存按钮，下方为当前 Tab 的表单 | `EventManage.vue` / `ContentEditor.vue` |

顶栏（`TopBar.vue`）展示页面标题、语言地球图标、当前用户，均为静态展示。

> **核心数据模型**：整个页面维护一份全局活动配置 `EventConfig = { tabs: Tab[] }`。
> 每个 `Tab` 内含三套语言内容（简体 / 繁体 / English），**Tab 结构在三语言间共享，内容各自独立维护**。

---

## 2. 模块 Tab 操作栏（左侧）

### 2.1 添加 Tab
- 点击右上角「**+ 添加**」即新增一个标签页，并自动选中、定位到该 Tab。
- 新 Tab 三语言的「传参值」默认填充：简体「新标签页」、繁体「新標籤頁」、English「New Tab」；其余字段为空。
- 新增无数量上限。

### 2.2 选中 Tab
- 单击列表中任意 Tab 即选中，右侧编辑区切换为该 Tab 的内容。
- 选中态以主题蓝边框 + 浅蓝底高亮。

### 2.3 拖拽排序
- 每个 Tab 左侧有拖拽手柄（⠿ 图标），按住可上下拖动调整顺序。
- **列表顺序即前台展示顺序**，拖拽实时生效。
- 拖拽过程中被拖项半透明显示。

### 2.4 删除 Tab
- 鼠标悬停或选中某 Tab 时，右侧出现红色「✕」删除图标。
- 点击后弹出二次确认弹窗：「确认删除「{传参值}」吗？删除后不可恢复。」
  - 确认 → 删除该 Tab（含三语言全部内容）。
  - 取消 → 不做任何变更。
- 若删除的是当前选中 Tab，则自动选中其前一个 Tab；删除后列表为空时编辑区显示空状态提示。

### 2.5 列表名称展示
- 左侧列表显示的是**当前语言下该 Tab 的「传参值」**。
- 切换语言后，列表名称会随之变化为对应语言的传参值；传参值为空时显示「未命名」。

---

## 3. 语言切换（编辑区顶部）

- 分段控件三选一：**简体中文 / 繁體中文 / English**，默认简体中文。
- 切换语言只影响右侧表单展示的内容版本，**不影响 Tab 列表结构本身**。
- 三语言内容相互独立：在简体下填写的内容不会同步到繁体 / English，需分别维护。
- 切换语言 / 切换 Tab 时，已填内容均保留，不会丢失（见第 6 节自动暂存）。

---

## 4. 内容编辑区字段规则

当前选中 Tab + 当前语言下，表单包含以下字段：

| 字段 | 是否必填 | 控件 | 规则 |
| --- | --- | --- | --- |
| 传参值 | **必填** | 单行输入框 | 最多 30 字符，带字数统计；同时作为左侧 Tab 名称 |
| 标题 | 选填 | 单行输入框 | 最多 100 字符，带字数统计 |
| 持续时间 | 选填 | 单行输入框 | 自由文本，建议格式如 `2026.06.01-2026.06.22` |
| 封面 | **必填** | 图片上传（**单张**） | jpg/jpeg/png，≤5MB，仅可上传一张 |
| 轮播区域图片 | 选填 | 图片上传（**多张**） | jpg/jpeg/png，≤5MB，可上传多张 |
| Markdown 正文 | 选填 | Vditor 编辑器 | 富文本 Markdown，支持正文内插图 |

> 字段必填校验在「保存并生效」时统一触发（见第 5 节），编辑过程中不做强校验。

### 4.1 封面上传（单张）
- 点击「上传」框选择本地图片，仅取第一张。
- 上传成功后以缩略图替换上传框（单张模式下不再显示上传入口）。
- 缩略图悬停出现删除按钮，删除后可重新上传。
- 格式 / 大小不符时给出错误提示，不写入。

### 4.2 轮播区域图片上传（多张）
- 点击「上传」框可一次选择多张图片，逐张校验后追加到列表。
- 每张缩略图悬停出现删除按钮，可单独删除。
- 提示文案：「若不上传图片，则前台自动全部显示下方 Markdown 内容。」
  即：**轮播图为空时，前台改为全屏展示 Markdown 正文**。

### 4.3 Markdown 正文
- 采用 Vditor 即时渲染（IR）模式，工具栏支持标题、加粗、列表、引用、代码、表格、插图、全屏等。
- 正文内插入图片走与封面 / 轮播图相同的统一上传接口。
- 切换 Tab / 语言时，编辑器内容随之同步，不会出现光标跳动或内容串台。

---

## 5. 保存并生效

- 位于编辑区右上角，点击后对**所有 Tab、所有语言**做统一校验与发布。
- **校验规则**（任一不通过即中断，并自动跳转定位到出错的 Tab + 语言并提示）：
  1. 至少存在一个 Tab，否则提示「请先添加至少一个标签页」。
  2. 每个 Tab 的每种语言「传参值」必填。
  3. 每个 Tab 的每种语言「封面」必须已上传。
- 校验全部通过后调用发布接口，按钮进入 loading；成功后提示「已保存并生效」。
- 「标题」「持续时间」「轮播图」「正文」均为选填，不参与校验。

---

## 6. 自动暂存（草稿）

- 编辑区任意内容变更后，**防抖 800ms** 自动暂存为草稿。
- 暂存期间编辑区右上角显示「自动保存中…」，完成后显示「已自动保存草稿」。
- 作用：切换 Tab / 语言、意外刷新等场景下内容不丢失；下次进入页面优先加载草稿。
- 草稿与「保存并生效」相互独立：草稿仅防丢失，**只有点击「保存并生效」才会对前台正式生效**。
- 页面初次加载数据时不会触发自动暂存。

---

## 7. 空状态与边界

| 场景 | 表现 |
| --- | --- |
| 无任何 Tab | 编辑区居中提示「当前暂无标签页，请点击左侧「+ 添加」创建一个 Tab。」 |
| 删除至列表为空 | 同上 |
| 传参值为空 | 左侧列表对应项显示「未命名」 |
| 图片格式/大小不符 | 顶部弹出错误提示，文件不写入 |
| 保存校验失败 | 自动切换到出错 Tab + 语言，并弹出对应错误提示 |

---

## 8. 接口对接说明

所有数据交互集中在 `src/api/event.js`，函数签名与返回结构已固定，对接时仅替换函数体：

| 函数 | 用途 | 建议接口 |
| --- | --- | --- |
| `getEventConfig()` | 读取活动配置（优先草稿） | `GET /api/mall/event/config` |
| `saveDraft(config)` | 自动暂存草稿 | `POST /api/mall/event/draft` |
| `publishEventConfig(config)` | 保存并生效 | `POST /api/mall/event/publish` |
| `uploadImage(file)` | 图片上传（封面/轮播/正文共用） | `POST /api/upload` |
